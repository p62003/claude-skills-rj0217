## 使用規則

1. **不要跳過「做得好的地方」** — 只列壞的扭曲實際風險畫面
2. **每個發現要有「嚴重度理由」** — 避免「我覺得這個 HIGH」的主觀判斷
3. **修法要具體可執行** — 不是「加驗證」，是「在 X 檔 Y 行加 `zod.string().regex(...)`」
4. **同業對照是**選填** — 沒資料就不寫，不要為了寫而編
5. **meta 學習是**選填** — 但 session 中真有洞察就記下，對未來 session 有用
